Choose the mission flow that fits your session

Manual start is always available. Agent-aware attach saves typing when Codex, Claude Code, Cursor, Gemini CLI, OpenCode, or Copilot has local session history.

manual
agentscope start "..." --agent codex

Best when you want exact control or detection is uncertain.

detect
agentscope agents detect

Find supported local agent logs and infer the latest usable task.

attach
agentscope attach --agent auto --apply

Write the inferred mission only after the dry run looks right.

monitor
agentscope monitor --agent auto

Detect context, show confidence, then open the live TUI.

enforce
agentscope check

Block protected paths and flag files outside the mission.

# initialize once per repo
agentscope init

# option A: precise manual session
agentscope start "Fix checkout button loading state" --agent codex

# option B: infer from current local agent logs
agentscope agents doctor
agentscope agents detect
agentscope attach --agent auto
agentscope attach --agent auto --apply

# watch and verify
agentscope watch
agentscope diff --problems
agentscope check

Local agent context readers

AgentScope reads local files only. It extracts likely user missions and filters out tool calls, patch hunks, metadata, login commands, timestamps, and app wrapper text.

AgentDefault sourceUser action
Claude Code~/.claude/projectsRun Claude once, then detect.
Codex CLI / app~/.codex/sessionsUse attach --agent codex or auto.
OpenCode~/.local/share/opencode/projectOverride path if storage moved.
Cursor~/.cursor/projectsUse local transcript detection when present.
Gemini CLI~/.gemini/tmpMetadata-only logs are ignored.
Copilot CLI~/.copilot/session-stateConfigurable because local formats vary.

Privacy model: detection reads local logs. It does not upload transcripts. The optional judge runs only when you run judge/check with judging enabled.

Missing sources are normal

You should not have to manually configure everything. Defaults work first, `doctor` explains gaps, and manual start is the reliable fallback.

01 Run doctor

agentscope agents doctor shows each checked path and explains that missing agents are not failures.

02 Override only when needed

Add agents.sources.<agent>.paths if your agent stores logs somewhere custom.

03 Stay safe by default

Auto-attach is off. Low-confidence missions do not silently rewrite your active session.

agentscope config show

agents:
  auto_detect: true
  auto_attach: false
  preferred:
    - codex
    - claude-code
    - cursor
  sources:
    codex:
      enabled: true
      paths:
        - "~/.codex/sessions"
        - "~/Library/Application Support/Codex/sessions"
