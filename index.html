What AgentScope does today

The current CLI is mission-based and Git-native. It does not yet read live Codex or Claude transcripts automatically.

01 Records an active mission

agentscope start writes `.agentscope/session.json` with the mission, agent label, baseline commit, and timestamp.

02 Checks changed files

agentscope check compares working tree changes with policy and the active session mission.

03 Watches while agents work

agentscope watch refreshes the current Git/filesystem view and shows file verdicts in a terminal UI.

04 Blocks protected paths

Deterministic glob rules protect `.env*`, secrets, auth paths, migrations, and any policy you configure.

05 Runs optional LLM judging

agentscope judge can use Ollama by default, with configurable Claude or OpenAI providers.

06 Reports for humans and CI

Use terminal output, JSON, markdown reports, annotated diffs, and a pre-commit hook.

Current CLI workflow

AgentScope works beside Codex, Claude Code, Cursor, Gemini CLI, OpenCode, and other tools because it watches the repo boundary rather than requiring an agent runtime.

setup
agentscope init

Create `agentscope.yaml` and local session storage.

mission
agentscope start "..." --agent codex

Declare the work you asked the agent to do.

monitor
agentscope watch

See changed files, line stats, and verdicts while work is happening.

verify
agentscope check

Run the deterministic policy check before commit or handoff.

judge
agentscope judge -m qwen3.5:2b

Ask a local model for a mission-fit assessment.

# initialize once per repository
agentscope init

# start a session for the agent work
agentscope start "Fix checkout button loading state" --agent codex

# monitor the current working tree
agentscope watch

# review and enforce policy
agentscope diff --problems
agentscope check
agentscope check --json

# optional model-based drift check
agentscope judge -m qwen3.5:2b
agentscope model list
agentscope config show

# optional git safety net
agentscope hook install

Agent-aware monitoring roadmap

The next layer is reading the current coding agent context directly, then keeping the active mission in sync without manual re-entry.

Planned agent adapters

  • Read local task context from Codex, Claude Code, Cursor, Gemini, OpenCode, OpenClaw, Hermes, Copilot, Droid, Pi, and custom agents.
  • Detect active agent, working directory, recent prompt, and confidence level.
  • Use manual sessions as fallback when no reliable agent context is available.

Planned watch upgrades

  • Auto-update `.agentscope/session.json` when the detected agent mission changes.
  • Show mission source and confidence in the TUI header.
  • Keep log reading local, best-effort, and opt-in/configurable.

Not shipped yet: native Stop hooks, MCP tools, plugin marketplace installs, wrapper commands like agentscope claude, and automatic per-turn agent transcript checks are roadmap items, not current production claims.
