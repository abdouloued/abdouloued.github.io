AgentScope is a scope firewall and audit layer for AI coding agents. It watches what they change and blocks policy violations — before they reach git.

Get Started → See it in action ↓
$ cargo install agentscope 📋
agentscope check

Why AgentScope?

Your agent fixed the bug. It also rewrote your auth module. AgentScope tells you — before you commit.

🚫

Policy Engine

Deterministic glob-based rules. Block .env, auth files, migrations — anything sensitive. No AI judgment needed.

🧠

LLM Judge

Ask a local LLM whether the changes match your mission. Runs on Ollama — no data leaves your machine. Pick any model.

📊

Live TUI Dashboard

Watch file changes in real time with bar charts, pie charts, health bars. Press 'j' to run the judge inline.

🔒

Git Pre-commit Hook

Install with one command. Every commit runs agentscope check. Policy violations block the commit automatically.

⚡

Agent-Agnostic

Works with any agent that writes to your filesystem. Claude Code, Codex, Cursor, Gemini CLI, Copilot, and more.

🔍

Full Audit Trail

Every session is logged. Review past sessions, export JSON for CI, share markdown reports in PRs.

Agent-Aware Monitoring Roadmap

From passive watcher to active observer. Here's where AgentScope is heading — honestly.

Today, AgentScope watches the filesystem. Tomorrow, it will read agent context — the task they were given, the model they're using, and what they think they're doing. All reading is local, best-effort, and opt-in.

Shipped

Filesystem-level monitoring

Git diff analysis, glob-based policy engine, LLM judge (Ollama / Claude / OpenAI), TUI dashboard with charts, pre-commit hooks, full audit trail.

Shipped

Model & config management

Claude Code / Codex-style commands: model list, model set, model test, config show, config set. Switch between Ollama models or cloud providers on the fly.

In progress

Read agent task context from local logs

Parse session transcripts from each agent's local storage to extract the original task/prompt. Auto-populate the AgentScope mission from what the agent was actually asked to do. Show source and confidence in watch mode.

Claude Code~/.claude/projects/<path>/*.jsonl
Codex CLI~/.codex/sessions/rollout-*.jsonl
OpenCode~/.local/share/opencode/project/<slug>/storage/
Cursor~/.cursor/projects/<hash>/agent-transcripts/
Gemini CLI~/.gemini/tmp/<hash>/chats/
Copilot CLI~/.copilot/session-state/
Privacy guarantee: All log reading happens locally on your machine. AgentScope never uploads transcripts, prompts, or session content anywhere. Each agent reader is opt-in and can be disabled in agentscope.yaml. If a log format changes or can't be parsed, AgentScope falls back silently to manual mission input.
local-only opt-in best-effort configurable
Planned

Auto-detect active agent

Detect which agent is running by checking for known process signatures, lockfiles, and local config directories (.claude/, .codex/, .cursor/). Automatically set the agent type in agentscope start without requiring --agent flag.

process detection lockfile heuristics
Planned

Mission inference & live update

When an agent starts a new task, AgentScope will detect the new session transcript, extract the user's prompt, and auto-update the active mission. The TUI will show inferred mission text with a confidence indicator ("inferred from Claude Code transcript, 92% match") so you always know the source.

JSONL parsing SQLite reading confidence scoring
Planned

MCP server for agent integration

Expose AgentScope as an MCP (Model Context Protocol) server so agents can natively query policy, check scope, and report their own mission — instead of AgentScope having to guess. Agents that support MCP tools will be able to call agentscope.check_scope and agentscope.report_mission directly.

MCP server stdio transport bidirectional
Planned

Plugin & skill system

A plugin architecture so the community can add new agent readers, custom policy rules, output formats, and notification hooks. Planned integrations include Slack/Discord notifications, GitHub PR comments, and OpenTelemetry trace export for enterprise observability stacks.

plugins community OpenTelemetry webhooks
Planned

Multi-agent session tracking

Track multiple agents working on the same repo simultaneously. Compare their actions, detect conflicts, and show a unified view of who changed what. Essential for teams running different agents on different tasks in the same codebase.
